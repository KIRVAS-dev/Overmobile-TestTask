1. Выбранный движок и почему:

Unity 6000.5.0f1, платформа WebGL

Для поставленной задачи сделать интерактивный рекламный ролик с 2.5D графикой, анимациями, звуком и возможностью оптимизации под браузер Unity очень хорошо подходит. К тому же - это единственный движок, с которым я работаю =)

-----------------------------------


2. Как собрать и запустить локально:

- Ссылка на live demo: https://kirvas-dev.github.io/Overmobile-TestTask
- Ссылка на репозиторий: https://github.com/KIRVAS-dev/Overmobile-TestTask

Перед клонированием установите и активируйте Git LFS (git lfs install) — без него не скачаются текстуры из Assets/_Project/.

Репозиторий Overmobile-TestTask -> проект открывать из папки Overmobile.

В редакторе запускать из сцены Bootstrap. Если в меню включить PlayFromBootstrap -> Enabled, то можно запускать из любой сцены (но в проекте их всего 2: Bootstrap и Core)

-----------------------------------


3. Какие оптимизации по размеру были применены:

Для оптимизации билда я использовал ассет собственной разработки - Build Preset Tool: https://assetstore.unity.com/packages/tools/utilities/buildpresettool-354574

Для каждой платформы применяются настройки, рекомендованные разработчиками Unity для повышения производительности билда и его размер на диске:

- Gzip
- DiskSizeWithLTO 
- Strip Engine Code 
- Managed Stripping Level: Minimal 
- Burst: Optimize For Size и др.

Также было применено платформенное сжатие текстур для WebGL - ASTC. Текстуры упакованы в атласы.

-----------------------------------


4. Что бы улучшил при наличии дополнительного времени:

- Можно было бы добавить заставку в начале и в конце игры
- Систему перемещений сделал бы не по фиксированным точкам маршрута, а с автовыбором пути 
- Можно было бы сделать больше эффектов камеры, анимаций, VFX и звуков 
- Сделал бы каждый запуск игры с уникальными расстановками противников их показателями силы
- Сделал бы полноценные бои между героем и противниками

Но в моем случае, я бы ответил наоборот: что бы я мог сделать, чтобы СОКРАТИТЬ время разработки: 

- Иметь шаблон проекта для быстрого прототипирования со всем необходимыми предустановленными пакетами и асетами. В итоге я сделал такой из данного проекта.
- Иметь готовые библиотеки игровых механик, которые можно переиспользовать и быстро добавлять в игру. Этот проект я старался делать таким образом, чтобы взять на будущее наработки.
- Много времени ушло на подготовку ассетов из представленных материалов, а также на подгонку таймингов, изингов анимаций и саундэффектов.
